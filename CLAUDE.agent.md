# 오디션 수집 에이전트 — 자동 수집·분류·학습 시스템

## 3. 에이전트 실행 파이프라인

```
[Phase 1: 수집]
    ├── 사이트 크롤러 (기존 9개 + 신규)
    ├── SNS 수집기 (인스타 해시태그, 네이버 카페)
    └── 기획사 직접 크롤링

        ↓ (원본 데이터)

[Phase 2: 분류] — 수집과 병렬 실행
    ├── 키워드 기반 1차 분류 (빠름, 토큰 0)
    ├── 규칙 기반 2차 분류 (정규식, 토큰 0)
    └── AI 3차 분류 (애매한 경우만, 토큰 최소화)

        ↓ (분류된 데이터)

[Phase 3: 저장]
    ├── Supabase upsert (중복 URL 기준)
    ├── 카테고리 태그 저장
    └── 수집 로그 기록

        ↓

[Phase 4: 학습/개선] — 주 1회 자동 실행
    ├── 분류 정확도 리포트
    ├── 소스별 수집률 분석
    ├── 토큰 사용량 분석
    └── 개선 사항 도출 → 규칙 업데이트
```

## 4. 분류 알고리즘 (3단계)

CATEGORY_KEYWORDS = {

    "singer": ["가수", "싱어", "보컬", "노래", "음원", "싱어송라이터"],
    "trot": ["트로트", "미스트롯", "미스터트롯", "국민가수"],

    "voice_actor": ["성우", "더빙", "나레이션", "보이스오버"],

```
- 제목 + 설명 텍스트에서 키워드 점수 합산
- 가장 높은 점수의 카테고리 배정
- **확신도 80% 이상이면 여기서 종료** → AI 호출 안 함

### Stage 2: 규칙 기반 보정 (비용 0)
- 출처 사이트 기반: `casting114` → 대부분 배우, `megaphone` → 대부분 모델
- 나이 조건 기반: "7~12세" → 키즈모델
- 급여 키워드: "페이", "출연료" → 엑스트라/보조출연 가능성
- 복수 카테고리 허용: "뮤지컬 배우" → [musical, actor]

### Stage 3: AI 분류 (최소 비용)
- Stage 1-2에서 **확신도 60% 미만인 경우에만** AI 호출
- Claude Haiku 사용 (가장 저렴)
- 배치 처리: 한 번 호출에 최대 10개 공고 동시 분류
- **프롬프트**:
```
다음 오디션 공고들의 카테고리를 분류해주세요.
카테고리: actor, model, idol, kids_model, singer, trot, photo_model, musical, theater, voice_actor, dancer, mc, extra, influencer

JSON 배열로 응답: [{"index": 0, "category": "actor", "sub": "드라마배우"}]

공고 목록:
{batch_items}
```

## 5. 토큰 비용 최적화 전략

### 현재 비용 구조
- 크롤링: 토큰 0 (직접 HTTP/Playwright)
- 분류: Claude API 호출 시에만 토큰 소비

### 최적화 원칙
1. **AI는 최후의 수단** — 키워드/규칙으로 90% 이상 분류 가능하게
2. **배치 처리** — 1건씩 호출하지 말고 10건 묶어서 1회 호출
3. **캐싱** — 동일 패턴의 공고는 이전 분류 결과 재사용
4. **Haiku 사용** — 분류는 단순 작업이므로 가장 저렴한 모델로 충분
5. **학습 루프** — AI가 분류한 결과를 규칙에 반영 → 다음에는 AI 호출 불필요

### 비용 절감 측정
```
수집 로그에 기록:
- total_collected: 수집된 공고 수
- classified_by_keyword: 키워드로 분류된 수
- classified_by_rule: 규칙으로 분류된 수  
- classified_by_ai: AI로 분류된 수
- ai_tokens_used: AI 토큰 사용량
- ai_cost_usd: 예상 비용

목표: AI 분류 비율을 전체의 10% 이하로 유지
```

```

기간: 2026-04-01 ~ 2026-04-07

[수집 현황]

- 신규 저장: 128건
- 중복 스킵: 214건
- 소스별: plfil(45) otr(38) vaudition(32) ...

[분류 현황]
- 키워드 분류: 298건 (87.1%)
- 규칙 분류: 32건 (9.4%)
- AI 분류: 12건 (3.5%)
- AI 토큰: 2,400 tokens ($0.003)

[카테고리 분포]
- 배우: 142건 (41.5%)
- 모델: 68건 (19.9%)
- 뮤지컬: 34건 (9.9%)
- ...

[개선 제안]
- "웹드라마" 키워드 추가 시 AI 호출 5건 절약 가능

### 자동 규칙 업데이트
1. AI가 분류한 결과를 수집
2. 반복되는 패턴 추출 (예: "~살 아동" → kids_model)
3. 키워드/규칙에 자동 추가
4. 다음 주기부터 해당 패턴은 AI 없이 분류
